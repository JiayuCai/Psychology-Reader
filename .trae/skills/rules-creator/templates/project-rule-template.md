# 项目规则模板

按需复制并替换以下占位内容。

适用于 `.trae/rules/*.md` 类型的项目规则文件。

```markdown
---
alwaysApply: false
description: "当用户需要……时，应用这条规则。"
globs:
  - "src/**/*.ts"
---

# 规则名称

## 适用范围

- 当前项目
- 目标目录或文件类型：`src/**/*.ts`
- 生效方式：智能生效 / 指定文件生效 / 始终生效 / 手动触发

## 规则内容

- 应……
- 优先……
- 禁止……

## 补充说明（可选）

- 若涉及多个无关约束，拆成多条规则文件
- 若只想对某个模块生效，优先放到对应子目录的 `.trae/rules/`
```

## 字段选择建议

- `alwaysApply: true`
  - 用于所有对话都必须遵守的项目级约束
- `alwaysApply: false` + `globs`
  - 用于仅对特定文件或目录生效的规则
- `alwaysApply: false` + `description`
  - 用于由 AI 根据场景判断是否调用的智能生效规则
- `alwaysApply: false`
  - 若不配 `globs` 或 `description`，通常适合作为手动触发规则使用

## 写法建议

- 一条规则只表达一组强相关约束
- 多用“应/优先/禁止/不要”这类可执行表述
- 避免“写得更好”“更专业”这类不可判定措辞
- 若规则会影响多个模块，先判断是否应提升为项目根规则
