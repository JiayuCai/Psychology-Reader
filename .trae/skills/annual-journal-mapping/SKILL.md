# Annual Journal Mapping

## 作用

这个 Skill 用于把某一年或某一卷期刊整理成一张可快速理解的“年度期刊脉络图”。

它不是逐篇精读，也不是简单目录复述，而是帮助用户从整卷层面提炼：

- 编辑视角：这一年为什么这样选题
- 年度主线：今年最重要的 `4-7` 条主题线索
- 子领域分布：大致落在哪些心理学子领域，各有多少篇
- 代表文章：每条主线由哪些文章支撑
- 阅读路径：如果只读少数几篇，应该先读什么

这个 Skill 特别适合：

- 年度综述期刊的入口阅读
- 想先建立全卷认知，再决定精读哪几篇
- 准备搭建“年度期刊地图”或“主题导航页”
- 需要 brief、结构化、可复用的期刊级摘要

## 何时使用

当出现以下情况时，优先调用本 Skill：

- 用户想了解“这一卷/这一年的重点是什么”
- 用户想读某本期刊的 `Introduction`，并关心编辑怎么看这一卷
- 用户想知道今年文章分别属于哪些子领域、各有多少篇
- 用户想得到“编辑视角 + 几条主线”的年度期刊概览
- 用户想从整卷层面安排后续阅读顺序
- 用户准备把年度期刊内容做成网页、数据库或知识地图

## 何时不要使用

以下情况不要调用本 Skill：

- 用户要的是单篇论文精读
- 用户只想看某篇文章的摘要或翻译
- 用户需要的是严格的书目计量分析或系统综述统计
- 用户要跨多个期刊做领域总图谱，而不是单卷期刊脉络
- 用户提供的信息不足以定位卷次、年份或目录页，且不允许补检索

## 输入清单

开始前尽量确认以下信息：

- 期刊名
- 年份或卷次
- 卷目录页链接、期刊主页链接，或至少一篇该卷文章链接
- 是否需要读取卷首 `Introduction`
- 用户更看重：
  - 编辑视角
  - 年度主线
  - 子领域分布
  - 阅读顺序建议
- 输出希望多 brief，还是允许稍作展开

如果用户没有指定格式，默认输出：

- 编辑视角
- 年度主线 `4-7` 条
- 子领域分布与篇数
- 建议阅读顺序

## 默认执行流程

### 1. 先定位整卷入口

- 优先找到该卷目录页或 volume page
- 提取整卷文章标题、作者、页码、摘要级简介
- 若目录页分页，必须补齐所有分页后再统计
- 在进入总结前，先做一轮卷次审计，至少确认：
  - 正式综述总数
  - 当前已收集条目数
  - 是否仍有缺失文章
  - `Introduction` 是否拿到正文、摘要或仅有目录元数据

### 1.5 硬性校验

- 在输出任何“编辑视角、年度主线、子领域分布、阅读建议”之前，必须先完成以下校验：
  - 已获得该卷全部正式文章清单
  - 已确认正式综述总数
  - 已区分 `Introduction` 与正式综述
  - 若用户要求编辑视角，已获取 `Introduction` 正文或可验证摘要
  - 子领域计数总和必须等于正式综述总数
- 若以上任一条件不满足：
  - 不得输出强判断式年度总结
  - 必须先报告缺口
  - 默认停在审计结果，而不是继续推断

### 2. 优先读取卷首 Introduction

- 若卷首 `Introduction` 可得，优先提取：
  - 编辑如何定义当下学科处境
  - 为什么这一卷的选题具有代表性
  - 编辑对方法、理论或时代背景的判断
- 若普通抓取拿不到正文，但浏览器态能看到正文，优先基于浏览器态 DOM 提取
- 若正文仍不可得，明确说明编辑视角只能根据目录与元数据推断
- 若用户明确要“编辑视角”或 `Introduction` 解读，但没有拿到引言正文或可验证摘要，则不得生成看似确定的编辑立场，只能报告证据不足

### 3. 建立文章清单

- 列出该卷全部正式文章
- 将 `Introduction` 与正文综述分开处理
- 不遗漏尾部分页或“每页显示数量”导致的隐藏文章

### 4. 归并为宽口径子领域

- 以用户可用为优先，不做过细分类
- 默认优先考虑这些宽口径：
  - 认知 / 认知神经 / 感知 / 注意 / 决策
  - 社会 / 人际 / 身份 / 群体 / 社会认知
  - 发展 / 教育 / 生命周期
  - 临床 / 健康 / 干预 / 风险
  - 人格 / 个体差异 / 基因组
  - 方法 / 测量 / 研究设计
  - AI / 计算 / 机器人 / 数字技术
- 每篇文章只放入最主要的一个归类；若明显跨界，可在主线部分再体现交叉性

### 5. 提炼年度主线

- 从整卷中提炼 `4-7` 条最有解释力的主线
- 主线不是机械按子领域列目录，而要回答：
  - 今年心理学最在意哪些问题
  - 哪些方向被新方法重写
  - 哪些现实或技术背景在给学科施压
- 每条主线至少配 `1` 篇证据锚点文章，可扩展到 `1-4` 篇
- 允许“一篇文章就构成一条主线”，前提是它确实代表该卷一个足够清晰的年度问题
- 每条主线都必须落回已核对过的正式文章标题，不能凭空概括

### 6. 压缩为编辑视角

- 用 `2-4` 句压缩卷首编辑立场
- 如果编辑引言可得，尽量基于原文
- 如果引言不可得，则明确标注这是根据目录与选题分布做出的推断
- 必须标记证据来源：
  - `introduction_fulltext`
  - `introduction_abstract`
  - `title_abstract_distribution`
  - `not_available`

### 7. 给出阅读路径

- 默认给出一种最省力的阅读顺序，例如：
  - 先读卷首 Introduction
  - 再读最能代表年度压力点的 `2-3` 篇
  - 最后按用户兴趣进入某条主线
- 若用户只想快速建立整体认知，优先推荐最能代表全卷脉络的文章

## 输出要求

默认输出应包含以下四层：

### 0. 审计结果

- `expected_article_count`
- `collected_article_count`
- `is_complete`
- `introduction_evidence_basis`
- `missing_items`（若有）

### A. 编辑视角

- 用一句或一小段概括编辑怎么看这一卷
- 重点写“今年为什么是这些方向”
- 若证据不是 `introduction_fulltext` 或 `introduction_abstract`，必须显式提示这是目录级推断，不是卷首原文立场

### B. 年度主线

- 提炼 `4-7` 条主线
- 每条主线说明：
  - 这条线在讲什么
  - 代表文章有哪些
  - 证据锚点最少可以是 `1` 篇

### C. 子领域分布

- 用宽口径分布说明这一卷主要覆盖哪些子领域
- 给出大致篇数
- 不要求做成严格统计图，但要保证计数完整、口径一致
- 必须校验 `sum(subfieldDistribution.count) = total_formal_articles`

### D. 阅读建议

- 如果只读 `3-5` 篇，建议先读什么
- 说明每篇在整卷里的作用

## 风格要求

- 优先 brief，不把目录逐篇扩写成冗长摘要
- 重点是“脉络感”，不是“信息堆砌”
- 多用“主线”“压力点”“方法升级”“选题信号”“编辑立场”这类结构词
- 少用空泛褒义词，如“很重要”“很有价值”
- 当判断基于卷首原文时，尽量说明；当判断基于目录归纳时，也要说明

## 失败策略

### 找不到整卷目录

- 先根据单篇文章反查卷目录页
- 若仍无法定位，暂停并说明缺口

### 目录页分页导致统计不完整

- 必须补抓所有分页，或使用更大 `pageSize`
- 若仍不完整，不要输出“篇数统计”式强判断
- 默认输出审计结果并暂停，不继续进入主线和分布总结

### Introduction 页面正文不可得

- 优先尝试浏览器态 DOM 提取
- 若仍不可得，退回到：
  - 目录级编辑视角推断
  - 并明确说明精度限制
- 若用户明确要“编辑视角”，而当前只有目录级材料，则必须先说明限制，再决定是否继续

### 子领域归类边界模糊

- 优先采用宽口径、少类别
- 保证口径一致，避免为每篇单独发明分类

### 用户同时要整卷脉络和单篇精读

- 先完成整卷脉络图
- 再建议对代表文章单独调用 `paper-deep-reading`

## 典型触发语句

- “帮我梳理这一卷期刊今年的重点”
- “先读这份期刊的 introduction，告诉我编辑怎么看”
- “按编辑视角 + 几条主线总结今年这卷”
- “把这一卷按子领域分一下，分别有多少篇”
- “我想先建立年度期刊脉络，再决定读哪几篇”
