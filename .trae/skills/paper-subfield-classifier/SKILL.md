# Paper Subfield Classifier

## 作用

这个 Skill 用于把一组论文、期刊卷次条目或年度文章清单，归入一套稳定、可复用的主题分类结构。

它不做单篇精读，也不做纯目录抄写，而是帮助用户得到可直接写入数据库或前端展示层的分类结果，重点生成：

- `primarySubfield`：每篇只给一个主归类
- `secondaryTags`：补充交叉主题、方法、应用场景或核心构念
- `subfieldRationale`：说明为什么这样归类
- `yearSummary.subfieldDistribution`：按年份或卷次汇总的子领域分布与概述

这个 Skill 特别适合：

- 年度综述期刊的卷次整理
- 构建论文地图或知识导航页
- 给前端展示页补齐分类字段
- 为后续抓取/分析流水线生成结构化 JSON

## 何时使用

当出现以下情况时，优先调用本 Skill：

- 用户要把一批论文分到若干子领域
- 用户要生成 `primarySubfield`、`secondaryTags`、`subfieldRationale`
- 用户准备做年度地图、分布图、主题卡片或筛选器
- 用户需要“宽口径但稳定”的分类口径，方便后续多年复用
- 用户已经有文章标题、摘要或元数据，希望先做结构化归类

## 何时不要使用

以下情况不要调用本 Skill：

- 用户要的是单篇论文精读
- 用户只需要翻译标题或摘要
- 用户要做严格书目计量学、引文网络或主题模型分析
- 用户手头没有足够元数据，且不允许补检索
- 用户要的是官方原始 taxonomy 列表，而不是工作型归类结果

## 输入清单

开始前尽量确认以下信息：

- 论文列表、期刊卷次页或至少文章标题 + 摘要
- 归类用途：
  - 前端展示
  - 本地 JSON 存储
  - 后续筛选器/分布统计
  - 阅读路线图
- 用户是否已经指定 taxonomy
- 需要按单年、单卷还是跨年输出
- 用户更偏好：
  - 宽口径少类别
  - 细口径多类别

如果用户没有明确指定，默认采用“宽口径、少类别、可多年复用”的策略。

## 默认执行流程

### 1. 先确认分类用途

- 如果用途是网页筛选、分布图、知识地图，优先使用宽口径分类
- 如果用途是精读档案或专家级整理，可在 `secondaryTags` 里补细粒度信息
- 不要一上来追求几十个细分类，先保证可用性和一致性

### 2. 建立稳定 taxonomy

- 优先复用用户已有 taxonomy
- 如果用户没有给，就根据文章集合归纳一套 `6-10` 个宽口径类别
- 类别命名应：
  - 可读
  - 稳定
  - 彼此区分清楚
  - 适合前端直接展示

优先使用以下归类原则：

- 先按“核心研究问题”而不是方法来分
- 方法、应用场景、具体构念优先放进 `secondaryTags`
- 一篇文章只给一个 `primarySubfield`

### 3. 为每篇文章选择 primarySubfield

- 先看题目与摘要最中心的问题意识
- 判断作者最主要在推进哪个研究脉络
- 如果文章明显跨界，仍然只选一个最主要的落点
- 不要为了“完整”把一篇文章拆进多个主类

判断优先顺序：

- 研究问题
- 理论脉络
- 应用语境
- 方法或数据来源

### 4. 生成 secondaryTags

- 默认给 `3-6` 个
- 优先提取：
  - 核心构念
  - 关键方法
  - 关键应用场景
  - 相关理论传统
- Tag 应短、稳、可复用，不写成长句

### 5. 写 subfieldRationale

- 用 `1-2` 句说明这篇为何归入该类
- 要解释“为什么在这里”，不要只改写标题
- 优先写：
  - 文章的核心问题
  - 为什么不是旁边那个相近类
  - 如果跨界，主落点为何仍选这里

### 6. 汇总年度分布

- 统计每个 `primarySubfield` 的篇数
- 对每个子领域写 `1-2` 句年度概述，说明这一年该领域在讲什么
- `Introduction` 或编辑导论单独标记，不混入分布统计

### 7. 做一次口径检查

输出前检查：

- 是否每篇都只有一个 `primarySubfield`
- 相似文章是否用了同一口径
- `secondaryTags` 是否太碎或太长
- `subfieldRationale` 是否真正解释了归类理由
- 年度分布是否排除了导论条目

## 输出字段规范

默认按以下结构输出：

### 单篇条目

```json
{
  "primarySubfield": "判断、决策与行为改变",
  "secondaryTags": ["理性", "推理", "决策偏差"],
  "subfieldRationale": "文章系统梳理理性研究中的判断、推理与决策问题，主问题明显落在判断与决策脉络。"
}
```

### 年度汇总

```json
{
  "yearSummary": {
    "totalEntries": 13,
    "totalArticles": 12,
    "subfieldDistribution": [
      {
        "name": "判断、决策与行为改变",
        "count": 3,
        "overview": "本年这一簇集中讨论理性、跨期偏差与行为改变机制。"
      }
    ]
  }
}
```

## 输出要求

- 优先输出可直接落库的结构化结果
- 分类名尽量保持短而稳，不要频繁改写同义词
- `primarySubfield` 面向筛选与统计
- `secondaryTags` 面向检索与补充表达
- `subfieldRationale` 面向人工复核
- 当判断主要基于摘要时，要控制分类力度，不做过细结论

## 失败策略

### 信息不足

- 若只有标题，没有摘要，先做粗分类
- 明确说明这是标题级归类，精度有限

### 分类边界模糊

- 先选择最能代表核心研究问题的主类
- 把交叉性放入 `secondaryTags`
- 不要为了“面面俱到”制造双主类

### 用户要官方 taxonomy

- 先说明当前输出是工作型 taxonomy
- 若用户需要官方学科分类，再单独补检索官方口径

### 跨年项目口径漂移

- 优先沿用历史类别名
- 新文章尽量并入旧框架
- 只有当旧框架明显失真时，才新增类目

## 风格要求

- 语气像做数据库归档的研究助理，不像泛泛摘要器
- 优先“口径一致”而不是“分类炫技”
- 多用“主落点”“交叉标签”“归类理由”“年度概述”这类结构词
- 少写模糊评价词，如“重要”“前沿”，除非说明其分类意义
