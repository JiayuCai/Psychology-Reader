# Paper Deep Reading

## 作用

这个 Skill 用于对跨学科的**单篇**学术论文做结构化精读，并按固定的 `L1-L5` 骨架输出可复用结果。

它不是泛泛摘要器，也不是文献综述地图工具，而是把一篇论文拆成以下五层：

- `L1` 作者为什么写、在纠什么偏
- `L2` 核心论点与论证链
- `L3` 关键构念的原定义、常见误用与作者 fix
- `L4` 对母学科的反哺、压力测试与边界修正
- `L5` **具体** pivotal references 及其角色

它适合：

- 年度综述或理论综述的深读
- 方法反思文章
- 跨学科桥接文章
- 用户明确要求 `L1-L5` 式结构化拆解
- 需要把单篇论文分析沉淀成可复用资产

## 何时使用

当出现以下情况时，优先调用本 Skill：

- 用户要求“精读”单篇论文，而不是只要摘要
- 用户要求按固定层次拆解一篇论文
- 用户要求梳理论文的论证结构，而不是只看结论
- 用户要求提炼“作者纠正了什么偏差”“提出了什么 fix”
- 用户要求从单篇论文中提取关键概念、关键争论和 pivotal references
- 用户希望把论文分析结果沉淀成可复用模板
- 用户给出 DOI、论文链接、PDF 或标题，并希望做深读

## 何时不要使用

以下情况不要调用本 Skill：

- 用户只需要一句话概括论文
- 用户只需要翻译标题或摘要
- 用户只想导出参考文献格式
- 用户需要的是多篇论文的系统综述或主题地图，而不是单篇深读
- 用户给出的材料不足以支持精读，且不允许补检索
- 用户要的是卷次地图、年度主线、子领域分布，而不是单篇结构化深读

## 输入清单

开始前尽量确认以下信息：

- 论文链接、DOI、标题，至少有一个
- 用户希望精读的是摘要级，还是尽量抓正文级
- 是否有用户指定的输出结构
- 是否需要保留术语原文
- 是否需要补抓 pivotal references
- 是否需要把文章放回某个更大的研究脉络中理解

如果用户没有指定结构，默认使用本 Skill 的 `L1-L5` 框架。

## 核心约束

- 单一职责：只做**单篇**论文深读，不扩展成多篇综述或年度地图
- 证据优先：正文级优先，摘要级次之，目录级最低
- 不得伪造：没有证据就不能写成确定判断
- 模板刚性：`L1-L5` 必须逐层输出，不得擅自改写字段含义
- `L5` 刚性：`L5` 必须是**具体文献**，不得用“传统 / 学派 / 路线 / 文献脉络”替代

## 执行前门槛

在开始输出前，先判断以下前提：

1. 是否已定位到单篇论文
2. 是否已拿到基本元信息：标题、作者、年份、期刊、DOI 或链接
3. 当前证据等级是什么：
   - `正文级`
   - `摘要级`
   - `目录级`
4. `L5` 所需的 references 证据是否足够：
   - 正文中的参考文献区
   - 文中明确点名的关键前史文献
   - 参考文献列表或 PDF 书目信息

若第 `4` 项不满足，`L5` 不得伪造，也不得改写成“传统/脉络版”。

## 默认执行流程

### 1. 获取论文信息

- 优先获取文章标题、作者、年份、期刊、DOI
- 若可访问正文，优先基于正文
- 若正文不可得，明确说明本次为摘要级或目录级精读
- 若正文部分可得，明确说明哪些判断基于正文，哪些只能保守推断

### 2. 识别文章类型

- 判断是理论综述、年度综述、方法论文、实证论文，还是跨学科桥接文章
- 根据类型调整“论证链”和“关键参考文献”的提取方式

### 3. 提炼作者的问题意识

重点回答：

- 作者为什么写这篇
- 作者对现有领域最不满的地方是什么
- 作者想纠正哪些常见误解、默认设定、方法依赖或研究偏差

### 4. 重建核心论点与论证链

- 用一句话压缩中心命题
- 将论证过程拆成 `3-6` 步
- 区分“作者明确说的”和“作者隐含假设的”
- 如果只能拿到摘要，不要伪装成完整论证链，必须控制论证力度

### 5. 提炼学科抓手

对每个关键概念，按以下模板输出：

- 原理论或原定义
- 常见简化、误用或工程化改写
- 作者的修正或 fix

这里的“学科抓手”可以是：

- 心理学构念
- 社会学概念
- 教育学理论
- 经济学模型
- 计算机领域的任务定义或评价指标
- 生物医学中的机制或测量框架

### 6. 提炼对学科自身的反哺

重点回答：

- 这篇文章反过来拷问了哪些老概念或经典假设
- 暴露了哪些理论边界、方法盲点、外部效度问题或定义模糊问题
- 对母学科提出了什么更高要求

### 7. 选择 pivotal references

默认提取 `3-5` 篇**具体文献**，优先选以下几类：

- 作为起点的奠基文献
- 作者明显在接续或修正的前史文献
- 搭桥作用最强的综述或理论文献
- 能代表作者 fix 的关键文献

不要机械按引用顺序选，要按“对理解本篇最关键”来选。

### 7.1 `L5` 硬性规则

- `L5` 每条必须包含：
  - `参考文献`
  - `角色`
  - `为什么关键`
- `参考文献` 必须是**具体文献**，至少达到以下之一：
  - 作者 + 年份
  - 文献标题
  - DOI
  - 能唯一指向某篇文献的稳定书目信息
- 禁止用以下内容替代具体文献：
  - “某某传统”
  - “某某路线”
  - “某某学派”
  - “某某文献脉络”
- 如果无法确认具体文献，必须写：
  - `L5 暂缺：当前证据不足以确认具体 pivotal references`
- 不得把“证据不足”偷偷改写成“传统版 L5”

### 8. 输出前做一次质量检查

检查以下几点：

- 是否只是摘要复述
- 是否讲清了“作者为什么要写”
- 是否讲清了“作者修正了什么”
- 是否讲清了“这篇对学科本身意味着什么”
- pivotal refs 是否真的 pivotal，而不只是随手列出
- `L5` 是否是具体文献而不是抽象传统
- 若 `L5` 不足，是否明确标为暂缺而不是模糊替代

## 默认输出结构

若用户未指定格式，默认输出以下五层：

### L1 问题意识

- 作者为什么写
- 作者纠什么偏
- 这篇文章试图把讨论从哪里推进到哪里

### L2 核心论点 + 论证链

- 核心命题一句话
- 论证链分步展开
- 必要时指出隐含前提与可质疑点

### L3 学科抓手

对每个关键概念按以下结构展开：

- 原理论或原定义
- 常见误用
- 作者 fix

### L4 对学科自身的反哺

- 哪些老概念的边界被拷问
- 哪些方法或定义问题被暴露
- 这篇论文逼母学科回答什么问题

### L5 Pivotal References

每篇尽量说明其角色：

- 奠基
- 前史
- 转折
- 搭桥
- 修正

每一条都必须是**具体文献**，不能是传统概括。

## 输出要求

输出时遵循以下原则：

- 不做“摘要改写型”回答
- 优先输出论证结构，而不是信息堆砌
- 尽量区分：
  - 作者原话层
  - 分析归纳层
- 对强判断要说明依据来自摘要还是正文
- 如果正文不可得，必须说明精度限制
- 若文章来自特定学科，优先保留关键术语原文
- 适合时可以建议下一步延伸阅读，但不要让建议盖过本次精读主体
- `L5` 不满足证据门槛时，必须显式标 `暂缺`

## 失败策略

### 正文不可得

- 退回到摘要级精读
- 明确告诉用户哪些判断更稳，哪些只是初步推断
- 优先保留：
  - 问题意识
  - 核心命题
  - 概念线索
  - 学科反哺
- 如果 `L5` 证据不足，直接标 `L5 暂缺`

### references 不可得或不足以确认具体文献

- 先额外尝试：
  - 参考文献区
  - 文中点名引用
  - PDF 或页面书目区
- 若仍不足：
  - 输出 `L5 暂缺：当前证据不足以确认具体 pivotal references`
  - 不得用“传统 / 学派 / 路线 / 脉络”代替
  - 必要时向用户说明缺口并建议补抓 PDF 或 references 区

### 文章信息不完整

- 先用用户给出的链接、标题或 DOI 补齐基本元信息
- 如果仍不足以精读，暂停并说明缺口

### 文章不是综述而是实证论文

保持 `L1-L5` 框架，但调整 `L2`：

- 加入研究问题
- 方法设计
- 主要发现
- 结论外推边界

### 用户要求过宽

如果用户同时要单篇精读、跨年主题地图和领域总综述：

- 先拆任务
- 默认优先完成单篇精读
- 再建议后续整合步骤

### 触发边界不清

- 如果用户既像是在要单篇深读，又像是在要卷次地图或多篇综述：
  - 先澄清目标
  - 未澄清前不要擅自按单篇精读输出

## 风格要求

- 语气像研究助理，不像百科摘要器
- 能压缩成一句的地方不要拖长
- 能指出“作者到底推进了哪一步”的地方，不要停留在“这篇讲了什么”
- 多用“纠偏”“推进”“重定义”“补边界”“压力测试”这类结构性表述
- 少用空泛褒义词，如“重要”“有价值”，除非说明重要在什么地方

## 模板文件

输出时优先参考同目录下的模板文件：

- `templates/output-template.md`
- `resources/review-checklist.md`

如果用户没有指定格式，直接按模板骨架展开。

- “不要只总结摘要，帮我拆论证结构”
